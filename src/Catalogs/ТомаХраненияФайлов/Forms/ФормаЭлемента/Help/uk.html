Опис властивостей тому зберігання файлів.

При створенні нового томи автоматично перевіряється доступ до зазначеного в полі Повний шлях до каталогу, у разі помилки виводиться відповідне повідомлення. Вже створеному тому не можна змінювати шляхи.

Для кожного тому необхідно вказати:

  • Найменування - повне найменування томи для зберігання файлів;
  • Код - заповнюється автоматично;
  • Повний шлях для сервера платформи під управлінням Microsoft Windows - напишіть шлях до каталогу тому для сервера під керуванням Microsoft Windows у форматі UNC - \\Ім'я комп'ютера\Найменування папки\Найменування файлу\;

або

    • Повний шлях для сервера платформи під управлінням Linux - напишіть шлях до каталогу для сервера під управлінням Linux;
  • Порядок - введіть ціле число, по умовчанню дорівнює 1. Заповнення томів здійснюється за порядком заповнення (за зростанням) до повного заповнення тому. Цю особливість можна використовувати, наприклад, для тимчасового вимкнення тому шляхом збільшення йому порядку заповнення.
  • Межа, МБ - можна вказати максимальний розмір всіх файлів на томі (квота тому). Якщо сумарний розмір файлів в томі перевищує встановлену квоту, то буде використаний наступний за порядком том. Якщо на жодному з доступних томів не буде достатньо місця для розміщення файлу, то інформація про це буде записана в журнал реєстрації і буде виведено повідомлення про помилку. Якщо квота встановлена рівною нулю, то при розміщенні файлів в томі вона не перевіряється;
  • Поточний розмір, МБ - проставляється програмою автоматично в міру наповнення тому, показує, наскільки том заповнений;
  • На закладці Коментар напишіть детальну інформацію про том.

Особливості заповнення поля Повний шлях для Linux

  • Якщо сервер платформи працює під Linux, то шлях до тому зберігання файлів можна вказати тільки локальний, наприклад, /home/path. Для того щоб використовувати UNC шлях, потрібно зробити на нього посилання. Для цього під правами адміністратора зробіть наступне:
    • виконайте команду виду mkdir folder, де замість "folder" напишіть ім'я свого каталогу. Наприклад, в каталозі /home виконайте команду mkdir shared - буде створено каталог shared з повним шляхом / home / shared.
    • виконайте команду виду mount -t cifs //server/share folder -o //domain/user, password=pwd, де
      - де //server/share - UNC шлях до іншого комп'ютера , folder - шлях до каталогу на комп'ютері Linux, наприклад \home\shared, //domain/user - ім'я домену/користувача, якщо необхідно, pwd - пароль користувача (якщо необхідно).
    • Якщо ім'я та пароль не потрібні, команда виглядає так: mount -t cifs //server/share folder.
  • Після цього в дорозі до того вказуємо folder, наприклад, /home/shared

Перевірка цілісності тому

  • Натисніть Перевірити цілісність, для того щоб сформувати звіт про цілісність даних в томі. Виводиться загальна кількість файлів і відсоток цілісних даних.

Вилучення зайвих файлів

  • Натисніть Вилучити зайві файли, для того щоб вилучити файли, які є на диску, але відомості про них в базі відсутні.
  • Передбачена можливість зберегти такі файли в зазначений каталог. За допомогою кнопки вкажіть Шлях каталогу для копіювання.

Перегляд списку файлів в томі

  • У кожному томі файли зберігаються в підкаталогах. На кожну дату створення версії файлу створюється свій підкаталог. Всі файли зберігаються з оригінальним іменем та розширенням, між ними додається номер версії, наприклад "Наказ.2.txt".
  • Для того щоб переглянути список файлів в томі, виконайте команду Файли в томі на панелі навігації томи.
