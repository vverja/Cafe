Редагування параметрів забезпечення

В даній формі можна встановити спосіб і метод забезпечення, які будуть застосовуватися для номенклатури по умовчанню.

Задання способу забезпечення


Одноразову зміну способу забезпечення, можна здійснити безпосередньо з робочого місця Формування замовлень за потребами на Кроці 3, у відповідній колонці.

Спосіб забезпечення може бути визначений індивідуально для кожної номенклатури.
Якщо спосіб забезпечення для номенклатури не заданий, то в момент формування замовлень за потребами потрібно в ручну уточнювати постачальника в полі Джерело забезпечення.

Задання методу забезпечення


Для номенклатури повинен бути обраний один з наступних методів забезпечення:

  • Замовлення під замовлення - при даному методі забезпечення в робочому місці Формування замовлень за потребами до замовлення буде пропонуватися тільки кількість, необхідна для відвантаження по замовленнях.
  • Підтримка запасу (min-max) - при даному методі необхідно вручну задати мінімальний і максимальний запаси товару. При зниженні запасу товарів до мінімальної кількості система буде пропонувати до замовлення кількість, рівну бажаному максимальному запасу. Статистика продажів при цьому методі не враховується.
  • Підтримка запасу (розрахунок за статистикою) - при даному методі буде запропоновано сформувати замовлення, коли поточного залишку товарів (за вирахуванням страхового) буде вистачати не більше, ніж на строк виконання замовлення за способом забезпечення, заданим для даного товару. При цьому денне споживання буде розраховане за статистикою минулих періодів. Кількість до замовлення за цим методом розраховується таким чином, щоб товару вистачило на заданий строк забезпечення або до зазначеної дати.
  • Підтримка запасу (розрахунок за нормою) - при даному методі буде запропоновано сформувати замовлення, коли поточного залишку товарів (за вирахуванням страхового) буде вистачати не більше, ніж на строк виконання замовлення за способом забезпечення, заданим для даного товару. Денна норма споживання при цьому методі задається вручну. Кількість до замовлення за цим методом розраховується таким чином, щоб товару вистачило на заданий строк забезпечення або до зазначеної дати.


Перемикач Потреби замовлень до відвантаження: визначає особливості розрахунку рекомендованої кількості до замовлення в обробці формування замовлень за потребами:

  • Забезпечувати за рахунок запасів - при розрахунку кількості для підтримання запасу враховується загальна кількість товару в наявності на складі (залишок) і загальна очікувана кількість товарів до надходження. При розрахунку кількості для забезпечення замовлень враховується вільний залишок товару в наявності на складі і частина очікуваних надходжень товару, не використаних для забезпечення замовлень. Відсутня кількість товарів пропонується до замовлення.
  • Забезпечувати незалежно від підтримання запасів - при розрахунку кількості для підтримання запасу враховується вільний залишок товару на складі і частина очікуваних надходжень товарів, не використаних для забезпечення замовлень. Для забезпечення замовлень пропонується до замовлення вся необхідне кількість товарів необхідна для відвантаження.


Страховий запас доступний для редагування тільки при методах забезпечення Підтримання запасу (розрахунок...) і дозволяє мінімізувати наслідки при несподіваних сплесків попиту, а також забезпечити запас товарів у разі зриву поставок.
