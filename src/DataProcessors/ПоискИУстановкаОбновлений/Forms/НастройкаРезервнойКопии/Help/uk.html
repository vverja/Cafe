Призначена для настройки резервного копіювання інформаційної бази при оновленні програми.

Перед початком оновлення програми рекомендується створювати резервну копію інформаційної бази. Резервна копія використовується для автоматичного відновлення інформаційної бази в разі нештатної ситуації.

Відкривається за посиланням Створювати тимчасову резервну копію ІБ і виконувати відкат за нештатної ситуації з вікна Пошук і установка оновлень.

Для створення резервної копії виберіть параметри:

  • Не створювати резервну копію;
  • Створити тимчасову резервну копію. У цьому випадку можна включити прапорець Відновлювати інформаційну базу, для того щоб можна було здійснити відкат при виникненні нештатної ситуації в процесі оновлення програми;
  • Створити резервну копію і зберегти в зазначеному каталозі. У полі каталог напишіть шлях для створення резервної копії, наприклад, С:\Резевное копіювання.

Після установки параметрів резервного копіювання натисніть ОК для повернення до установки оновлення.
