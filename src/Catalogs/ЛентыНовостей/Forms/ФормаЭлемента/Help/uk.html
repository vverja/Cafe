Редагування стрічки новин

Основні дані

  • Найменування- довільне найменування стрічки новин, видиме користувачам
  • Код- код стрічки новин, який використовується підсистемою отримання новин
  • Обов'язковість- можливість відключення стрічки новин користувачем. Обов'язкові стрічки новин користувач не зможе відключити і буде отримувати по ним новини завжди.
  • Видимість- тут задаються виключення видимості стрічки новин. Стрічка новин може бути по умовчанню видима всім, крім деякого списку користувачів. І навпаки- стрічка новин по умовчанню може бути від усіх прихована, іі буде видно тільки певним користувачам.
  • Коментар- довільний текст

Закладка "Настройки завантаження новин"

  • Це локальна стрічка новин- ознака, що стрічка новин створена вручну, і заповнюватись вона буде в самій інформаційній базі (вручну, користувачами з роллю РедакторНовостей або програмно)
  • Протокол- протокол зв'язку (для випадку, якщо стрічка новин не локальна). Можливі значення: http, https, file
  • Сайт- ім'я сайту, з якого буде проводитися читання новин у форматі atom або rss. Для випадку, якщо стрічка новин не локальна і протокол не file
  • Ім'я файлу- ім'я файлу на сайті (або в операційній системі), звідки будуть регулярно зчитуватися новини. Для випадку, якщо стрічка новин не локальна.
  • URL- результуюче посилання на файл з новинами (результат додавання Протокол, Сайт і Ім'я файлу). Якщо перейти за цим посиланням, то можна подивитися в браузері найактуальніші новини.
  • Підключення- Варіант авторизації. Зазвичай всі стрічки новин відкриті для всіх. Але в деяких випадках доведеться ввести логін і пароль. Це налаштовується в цьому пункті. Можливі значення:
    • Без логіна / пароля- логін і пароль для завантаження новин не потрібно (це варіант за замовчуванням)
    • Введено вручну (однаковий для всіх)- логін і пароль необхідно ввести вручну (він буде однаковий для всіх областей даних)
    • Введено вручну (різний для всіх)- логін і пароль необхідно ввести вручну (він буде відрізнятися для кожної області даних)
    • Значення логіна / пароля з Веб-ІТС - значення логіна і пароля будуть взяті з настройок доступу до Інтернет-підтримки користувачів
  • Логін- логін для доступу до стрічки новин, для випадку, коли Підключення- Введено вручну
  • Пароль- пароль для доступу до стрічки новин, для випадку, коли Підключення- Введено вручну
  • Завантаження бінарних даних- можна вимкнути для зменшення трафіку. Настійно не рекомендується це робити, тому що в іншому випадку перестануть завантажуватися картинки, вкладені в Новини

Закладка "Частота оновлення"

На цій закладці налаштовується частота оновлення стрічки новин з інтернету (у разі, якщо це не локальна стрічка новин).

Якщо вибрати Оновлювати тільки вручну, то підсистема читання новин перестане автоматично завантажувати новини в цю стрічку новин і доведеться оновлювати стрічку новин тільки вручну, за допомогою обробки Керування настройками і оновленням для новин (для відкриття потрібні адміністративні права)

Закладка "Настройка відборів"

Якщо стрічка новин дозволяє налаштовувати категорії, то на цій закладці можна налаштувати, з якими категоріями користувачі зможуть налаштовувати відбори, а також встановити значення відборів, які користувачі не зможуть змінити.

Для локальних стрічок новин список категорій налаштовується на цій закладці.

У стрічках новин список категорій змінити не можна.

Закладка "Редагування новин"

Цю закладку видно тільки у разі, коли стрічка новин є локальною (тобто коли новини не завантажуються в неї автоматично, а створюються користувачами з роллю РедакторНовостей).

Тут можна налаштувати список користувачів- редакторів новин, яким дозволено редагувати новини в цій локальній стрічці новин.

Закладка "Статистика"

На цій закладці можна подивитися загальну кількість новин по цій стрічці новин, а також подивитися дату найсвіжішої новини. Також можна скинути збережену дату останнього завантаження новин, щоб ініціювати повторне завантаження всіх новин.
