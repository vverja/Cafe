Сегмент клієнтів

Довідник призначений для зберігання списку всіх сегментів клієнтів, визначених у підприємстві.

Сегмент-це набір елементів, що задовольняють правилами, визначеним для сегмента. Він надає інформацію щодо елементів цього набору у виді, що відповідає цілі створення сегмента. Сегменти допускають перетину (один елемент може входити в кілька сегментів). Правила формування і форми представлення сегментів можуть використовувати довільну інформацію, що зберігається в системі.

Доступність довідника визначається функціональною опцією Сегментів клієнтів в розділі НДІ та адміністрування - CRM і маркетинг-Маркетинг.

За допомогою сегмента клієнтів проводиться додаткова Класифікація клієнтів, яка використовується при визначенні правил взаємодій і типових умов продажів, а також при проведенні маркетингового аналізу.

Поділ клієнтів на сегменти дозволяє:

  • Реєструвати різні типові умови продажів (ціни, знижки) для кожного сегмента клієнтів за допомогою типових оферт.
  • Проводити по кожному сегменту клієнтів окремі маркетингові акції.
  • Використовувати сегмент клієнта як розріз або відбір в різних аналітичних звітах.

Реєстрація сегмента клієнта
  • Найменування- текстове найменування сегмента. Найменування сегмента показується в списках для вибору і використовується для швидкої ідентифікації сегменту клієнтів.
  • Опис - детальний опис призначення сегмента клієнтів, правил його формування.
  • Спосіб формування- спосіб формування сегмента.


Після вказівки всіх основних параметрів натисніть на кнопку Записати. Настройка правил формування сегмента проводиться тільки після того, як визначені основні параметри сегмента (зокрема спосіб формування сегмента).

Після запису інформації про сегмент змінити спосіб формування сегмента не можна. Динамічні сегменти використовуються тільки для аналізу інформації. Динамічні сегменти не можна використовувати при визначенні умов продажу (оферта), умови щодо знижок, цін і т. д.

Автоматична перевірка виконання умов для входження в сегмент


Можливість передбачена для сегментів зі способом формування Періодично оновлювати. Якщо при введенні нового клієнта необхідно перевіряти приналежність його до сегменту, то в картці сегмента слід встановити прапор Перевіряти на входження при створенні нового клієнта.

Приклад. Сегменти клієнтів розділені по бізнес-регіонах. Для кожного бізнес-регіону настроєний свій сегмент з відбором по конкретному бізнес-регіону. При введенні нового клієнта і зазначення бізнес-регіону, він буде автоматично віднесений до певного сегменту.

Визначення правил формування сегмента
  • У картці сегмента клієнтів натисніть на кнопку Редагувати.
  • Буде відкрита форма настройки правил формування сегмента.
  • Буде завантажена основна схема, за допомогою якої може бути проведена настройка правил формування сегмента.


При настройки правил формування сегмента використовується система компонування даних. Користувачеві надається спрощена схема настройки. За допомогою спрощеної схеми настройки користувач може редагувати наступні настройки правил формування сегмента.

  • Проводити відбір даних відповідно до параметрів, які визначені схемою компонування сегмента (Відбір).
  • Задавати значення параметрів, якщо це передбачено схемою компонування сегмента (Параметри).


При формуванні нового сегмента завантажується основна схема компонування даних для формування складу сегмента клієнтів. У поставку програму входять додаткові схеми настройки, які поставляються у виді шаблонів.

Редагування складу сегмента
  • У панелі навігації форми сегмента виберіть гіперпосилання Склад сегмента.
  • З'явиться список, в якому представлені дані поточного сегмента.
  • Набір полів, що відображаються в списку, визначається настройкою правил формування сегмента.
  • Використовуючи кнопку Оновити можна оновити дані в складі сегмента відповідно до поточних даних інформаційної бази.

Редагування даних у списку складу сегмента можливо тільки в тому випадку, якщо для сегмента задано спосіб формування Формувати вручну. Для додавання інформації про нових клієнтів в список використовується кнопка Додати в сегмент, для видалення клієнта зі списку - Видалити із сегмента.

Визначення списку конкурентів


З будь-яким типом сегментів можна пов'язати список конкурентів, які працюють з тим же сегментом. Інформація про конкурентів заповнюється в табличній частині Конкуренти.

Аналіз даних по сегменту клієнтів
  • У панелі навігації картки сегмента виберіть пункт Продажу.
  • Буде сформований звіт, в якому буде показана динаміка продажів товарів клієнтам, що входять в сегмент, розподілена по місяцях.

Аналіз клієнтів, що відносяться до інших сегментів


Для аналізу використовується звіт Перетини сегмента клієнтів (гіперпосилання Перетини сегмента), в якому показуються перетини поточного сегмента з усіма іншими. При цьому поточний сегмент і аналізовані сегменти повинні бути нединамічними. Звіт викликається з панелі навігації картки сегмента. У звіті відображається згрупована за сегментами інформація.

  • Найменування сегмента.
  • Список клієнтів, за якими сегмент перетинається з аналізованим сегментом.

Аналіз стану взаєморозрахунків
  • У панелі навігації форми сегмента виберіть гіперпосилання Відомість розрахунків.
  • Буде сформований звіт Розрахунки з клієнтами з відбором по тих клієнтам, які включені в сегмент.

Аналіз продажів по сегменту


Для аналізу використовується звіт Динаміка продажів доступний за гіперпосиланням Звіти - Продажі по сегменту.
