Призначена для оцінки продуктивності роботи програми.

У програмі передбачені вбудовані засоби для збору і аналізу даних про продуктивність роботи за APDEX, яка є міжнародним стандартом оцінки продуктивності інформаційних систем.

Для того щоб оцінювати продуктивність, необхідно на початку роботи програми заповнити список Ключові операції.

Оцінка продуктивності програми за методикою APDEX складається з наступних основних етапів:

  • Визначення пріоритету для кожної ключової операції, передбаченої програмою;
  • Визначення цільового значення часу виконання кожної ключової операції;
  • Збір інформації про час виконання кожної ключової операції;
  • На підставі зібраних даних - формування оцінки продуктивності та інтерпретація отриманих значень.

До перегляду і оцінки результатів вимірів продуктивності програми можна перейти по команді Показники продуктивності розділу Адміністрування - Підтримка та обслуговування.

Таблиця

Показники продуктивності виводяться в таблицю, яка складається з колонок:

  • Ключова операція - найменування ключовий операції. У таблиці перераховані всі ключові операції, передбачені в програмі. Вони виводяться зі списку Ключові операції. Крім того, є напередвизначена ключова операція Загальна продуктивність системи, яка по умовчанню виводиться лише в таблицю.
    • За допомогою кнопок Додати і Вилучити можна сформувати список ключових операцій в програмі, за якими буде здійснюватися оцінка продуктивності.
    • Після того як ключова операція додана в таблицю, вона стає недоступною в списку для вибору (тобто одна операція може бути додана в таблицю один раз).
  • Пріоритет - виводиться зі списку Ключові операції.
    • Використовуючи кнопки Перемістити вверх і Перемістити вниз, можна переміщувати ключові операції в списку.
    • При переміщенні ключовий операції в списку в неї автоматично змінюється Пріоритет.
    • Сама верхня операція в списку має найвищий пріоритет, найнижчий запис - найнижчий. Таким чином, змінюючи розташування ключовий операції в списку, можна настроїти пріоритет для кожної операції.
    • Запис Загальна продуктивність системи завжди розташовується на початку списку, не має пріоритету і служить для оцінки продуктивності програми в цілому.
  • Цільовий час - виводиться зі списку Ключових операцій. Поле є обов'язковим для заповнення. В разі потреби може бути заповнене прямо в таблиці. Оцінка продуктивності буде будуватися щодо цього значення. Тобто при зміні цільових значень часу будуть змінюватися і оцінки продуктивності. Цільовий час вказується в секундах.

Настройка періоду

  • Перед проведенням оцінки продуктивності потрібно задати період, за який буде проводитися оцінка продуктивності.
    • Для цього введіть Дату початку та Дату закінчення періоду в форматі Дата-час. За допомогою кнопки виберіть дату з календаря, в разі потреби змініть час (по умовчанню дорівнює 0:00:00).
  • Установкою поля Крок регулюється кількість результатів, які будуть відображатися в періоді. Наприклад, якщо в якості періоду встановити день, а в якості кроку - годину, то у вікні буде виведено 24 результату (для кожної ключової операції). При цьому результат за годину буде розрахований за спеціальною формулою за методикою APDEX. Якщо протягом години операція не виконувалася в програмі, то у відповідній комірці таблиці буде нуль. Заміри будуть зроблені під час виконання зазначених ключових операцій, наприклад, при проведенні документа.
  • Результати замірів будуть показані у вікні Оцінка продуктивності на вкладках Таблиця і Діаграма.

Оновлення таблиці

  • Список не є динамічним. Для того щоб оновити дані в таблиці і діаграми, після того як в полях Період та/або Крок встановлені нові значення, можна скористатися командою Оновити показники продуктивності.

Обчислення цільового часу

  • Виділіть потрібну ключову операцію (крім операції Загальна продуктивність системи), виконайте команду контекстного меню по правій кнопці миші Обчислити цільове час з APDEX. Обчислення можливе, якщо за ключовою операцією були проведені виміри часу.

Відбір показників

Для відбору показників продуктивності натисніть Встановити відбір, за допомогою перемикача виберіть:

  • В полі Відібрати-- Краще чи Гірше;
  • для порівняння виберіть оцінку по APDEX: Добре, Задовільно, Погано;
  • в полі Умова автоматично проставляється цифрове вираження за APDEX, яке відповідає оцінкам.
  • Для початку відбору натисніть ОК.

Історія виконання ключової операції

  • Для того щоб переглянути Історію виконання ключової операції, використовуйте подвійне натиснення кнопки миші по значенню в таблиці. Виводиться період виконання замірів, зазначених у таблиці, параметри ключовий операції, заповнені зі списку ключових операцій, а також кількість вимірів. В таблиці вимірів вказується інформація по кожному виміру: Тривалість, Час закінчення, Користувач.

Діаграма

  • Для більшої наочності на вкладці Діаграма оцінки, отримані в таблиці представлені у вигляді графіків, де один графік відповідає одній ключовій операції.

Експорт вимірів продуктивності

  • Отримані виміри продуктивності програма дозволяє автоматично в фоні вивантажувати в зовнішні файли, наприклад, для виконання аналізу продуктивності в іншій програмі.
  • Для того щоб настроїти вивантаження цих даних, натисніть Ще - Настройка. Сформовані зовнішні файли, а також файли, що містять настройки, при яких були отримані ці виміри, автоматично упаковуються в ZIP-архів.
  • Для того щоб виконати експорт вимірів продуктивності вручну, натисніть Ще - Експорт даних. Файл із вимірами продуктивності можна відкрити або зберегти. Для збереження файлу виберіть шлях до папки на комп'ютері.
