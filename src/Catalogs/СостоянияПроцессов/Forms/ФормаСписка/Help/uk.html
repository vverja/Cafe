Стани процесів

Довідник призначений для зберігання списку етапів процесів продажів, які визначені для управління угодами з клієнтами.

У списку етапів процесів продажів відображається набір напередвизначених етапів, який обумовлений бізнес-процесами управління угодами, які застосовуються на підприємстві. Напередвизначені етапи користувач може використовувати і для угод, що мають тип з ручним переходом по етапах.

У довіднику можна настроїти послідовність виконання етапів. Ця послідовність буде використовуватися при оформленні угод з ручним переходом по етапах. При використанні бізнес-процесів для управління угодами послідовність виконання етапів строго визначається тим бізнес-процесом, який визначений для управління угодою. Як приклад бізнес-процесу в конфігурації застосовується бізнес-процес Типовий продаж.

Формування списку етапів


У списку етапів процесів продажів відображається набір напередвизначених етапів, який формується на підставі даних довідника Стани процесів. Список етапів процесів продажів може бути доповнений користувачем відповідно до потреби відображення збудованих на підприємстві процесів продажів. Надалі, для аналізу ефективності збудованих на підприємстві процесів продажів можна буде використовувати аналітичні звіти, наприклад воронку продажів.

Додавання етапу

  • У списку етапів процесів продажів натисніть на кнопку Створити.
  • У новому етапі процесу продажу вкажіть найменування етапу (поле Найменування) і текстовий опис етапу (поле Опис).
  • Натисніть на кнопку Записати і закрити.


Після введення інформації про новий етап необхідно визначити місце даного етапу в процесі продажу, тобто визначити послідовність етапів процесу продажу.

Настройка послідовності виконання етапів


Послідовність етапів виконання продажів налаштовується у формі списку Етапи процесів продажів. Послідовність визначає в якому порядку процес продажів буде рухатися по етапах. Для настройки послідовності використовуйте елементи управління в командній панелі списку (Перемістити елемент вверх, Перемістити елемент униз).

Послідовність етапів процесів продажу, налаштована в списку, надалі використовується при зазначенні етапів процесу продажів у тих видах угоди, які використовують ручний перехід по етапах. Порядок виконання етапів за угодами, які використовують бізнес-процеси, визначається тією послідовністю етапів, яка визначена при розробці бізнес-процесу.
