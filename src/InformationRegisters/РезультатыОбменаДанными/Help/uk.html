Призначений для перегляду статистики за результатами синхронізації даних.

Результати синхронізації даних у вигляді посилання можна побачити в списку Синхронізації даних, наприклад, на робочому столі програми. В залежності від результатів посилання приймає значення Попереджень немає (у разі успішної синхронізації даних) або Попередження (при цьому в дужках вказується кількість попереджень). Для відкриття списку результатів синхронізації даних пройдіть за посиланням.

При виконанні синхронізації даних можливе виникнення проблемних ситуацій, що вимагають втручання користувача. У колонках списку виводиться необхідна інформація про дані, щодо яких виникли проблеми та причину, по якій вони не були прийняті. Причина коротко зазначається у спеціальній колонці, при цьому по виділеному рядку в нижній частині списку виводиться докладне формулювання, із зазначенням проблемних полів списку або документів.

Відбори

Для зручності перегляду передбачені відбори:

  • Виберіть Період, за який видавати дані;
  • У полі Відправник Виберіть зі списку інформаційну базу, яка вас цікавить;
  • У полі Причина напишіть причину помилки (даний відбір неможливий для вкладки Конфлікти).

Проблемні ситуації можна умовно розділити на наступні категорії, які в списку знаходяться на окремих вкладках. У заголовку кожної вкладки в дужках вказується кількість відповідних проблемних ситуацій:

  • Непроведені документи

  • У цьому списку показуються документи, які не вдалося провести з яких-небудь причин. Для вирішення проблеми з проведенням документа рекомендується відкрити документ та усунути недоліки, що заважають його проведенню (це може бути, наприклад, недостатній залишок по товару). Внизу списку відображається причина неможливості автоматичного проведення.
  • Для того щоб спробувати виправити помилки в обраних документах, натисніть Змінити виділені.
  • Після усунення помилок можна Провести документ.
  • Для того щоб відкрити документ, натисніть Ще - Змінити.
  • Для приховування документа з цього списку без усунення проблеми з проведенням натисніть Ігнорувати.
  • Для того, щоб показати зігноровані документи, натисніть Ще - Показувати зігноровані.
  • Для повернення зігнорованого документа в список натисніть Ще - Не Ігнорувати.

Незаповнені реквізити

  • Містить список незаповнених реквізитів елементів довідників. Для виправлення необхідно відкрити елемент довідника (це можна зробити подвійним натисненням кнопки миші на ньому і заповнити обов'язкові для заповнення поля. Після чого необхідно записати елемент.
  • Для того щоб спробувати заповнити реквізити в обраних елементах, натисніть Змінити виділені.
  • Для того щоб відкрити елемент, натисніть Ще - Змінити.
  • Для вилучення елемента довідника з цього списку без усунення заповнення обов'язкових реквізитів натисніть Ігнорувати.
  • Для того, щоб показати Зігноровані елементи, натисніть Ще - Показувати зігноровані.
  • Для повернення зігнорованого елемента в список натисніть Ще - Не ігнорувати.

Конфлікти

  • При налаштованій синхронізації даних між двома програмами виникають ситуації, коли одні і ті ж дані змінюються одночасно (у проміжок між виконанням сеансу синхронізації даних) в обох програмах. Це призводить до виникнення двох ріних версій однакових даних у цих програмах. Така ситуація називається конфліктом зміни даних.
    Конфлікти зміни даних вирішуються (вибирається одна з версій) автоматично, проте можна змінити результат автоматичного вирішення конфлікту. Для цього виділіть елемент списку.
  • У рядку внизу списку виводиться метод вирішення конфлікту, який був обраний програмою.
  • Для виявлення відмінностей між двома версіями натисніть Показати відмінності.
  • Для прийняття рішення, яку версію прийняти, можна переглянути обидві версії. Для цього натисніть Ще - Показати версію, створену в цій інформаційній базі або Ще - Показати версію, отриману з іншої інформаційної бази.
  • Для перегляду елемента списку, натисніть Ще - Змінити дані.
  • Після прийняття рішення про версії, яка повинна бути збережена, натисніть Підтвердити для збереження результату автоматичного вирішення конфлікту або Переглянути для зміни результату розгляду конфлікту на протилежний. Після цього елемент стає неактивним, йому присвоюється ознака Розглянуто і по умовчанню він приховується зі списку.
  • По умовчанню розглянуті конфлікти приховуються зі списку. Для того щоб показувати їх теж, натисніть Ще - Показувати розглянуті конфлікти.
  • Для того щоб заново перевірити розглянуті конфлікти і що-небудь змінити, натисніть Ще - Зняти ознаку "Розглянуто".

Неприйняті за датою заборони

  • У цьому списку відображаються дані, які були відхилені із-за встановленоъ заборони на завантаження даних минулих періодів.
  • Для порівняння відхилених даних з поточними даними в програмі натисніть Показати відмінності (при наявності даних у поточній програмі).
  • Для того щоб відкрити елемент, натисніть Ще - Змінити.
  • Для перегляду іншої версії натисніть Ще - Показати неприйняту версію.
  • Для отримання даних, незважаючи на встановлену заборону завантаження даних, натисніть Прийняти версію.
  • Для видалення рядка з списку без прийняття рішення натисніть Ігнорувати.
  • Для того, щоб показати Зігноровані елементи, натисніть Ще - Показувати зігноровані.
  • Для повернення зігнорованого елемента в список натисніть Ще - Не ігнорувати.
