Призначена для встановлення дати заборони зміни (завантаження) даних.

Заборони завантаження даних можуть не використовуватися. У цьому випадку доступно тільки вікно настройки дат заборони зміни даних.

Встановлення дати заборони зміни даних та різні настройки, пов'язані з нею, виконуються за відповідною командою в розділі НДІ та адміністрування - Підтримка та обслуговування.

Встановлення дати заборони завантаження даних виконується за відповідною командою в розділі Синхронізація даних.

Про заборону зміни (завантаження) даних

  • Іноді при роботі з програмою потрібно встановити заборону зміни будь-яких даних до певної дати. Наприклад, після здачі квартальної бухгалтерської звітності в податкову інспекцію зазвичай встановлюється заборона зміни тих даних, які використовувалися для формування цієї звітності, тобто дані за останній день звітного періоду.
  • Заборона завантаження, на відміну від заборони зміни, може бути встановлений для даних, які повинні змінюватися тільки в одному вузлі інформаційної бази, а також за правилами, які випливають з вимог синхронізації даних, а не з типових бізнес-процесів організації. Наприклад, рішення редагувати дані по деяким організаціям лише у центральному офісі (у центральному вузлі), для того щоб технічно забезпечити закріплення відповідальності за ці дані за конкретним співробітником центрального офісу, є підставою заборони завантаження даних з інших сайтів.

Прояв встановленої заборони

  • Якщо дата заборони зміни даних встановлена, то при спробі внести зміни в інформаційну базу до дати зміни даних користувачу виводиться повідомлення про неможливість зміни даних.
    • Якщо в закритому періоді є об'єкти, помічені на видалення, то їх видалення буде неможливо.
    • Якщо дата заборони зміни не встановлена, то обмеження не діють.
  • Якщо дата заборони завантаження даних встановлена, то при спробі отримати дані в програму до дати заборони завантаження всі заборонені для одержання дані будуть пропущені, відомості про них будуть записані в журнал реєстрації, а дозволені дані будуть отримані.

Встановлення дати заборони

  • У програмі передбачена можливість встановити або відключити дату заборони змін (дату заборони завантаження даних). Для цього в полі Встановленнядати заборони виберіть відповідне значення:
    • Немає заборони зміни даних-в цьому випадку відключаються всі встановлені раніше дати заборони.
    • Для всіх користувачів (Для всіх інформаційних баз). При цьому стають доступними поля Спосіб зазначення дати заборони і Дата заборони.
    • По користувачах (По інформаційних базах). Цю можливість можна використовувати, якщо дату заборони необхідно встановити для конкретного користувача або групи користувачів (для конкретної інформаційної бази), або різним користувачам (інформаційним базам) необхідно встановити різні значення дати заборони.
      • При цьому стає доступним список, в який можна додавати користувачів/групи користувачів (інформаційні бази) і настроїти для кожного з них своє значення дати заборони.
      • Список складається з колонок:
        • Користувач, група користувачів (Програма: інформаційна база - найменування програми та інформаційної бази для цієї програми).
        • Коментар - напишіть додаткову інформацію про користувача (про інформаційну базу).
      • У списку за замовчуванням завжди присутній елемент <Для всіх користувачів> (<Для всіх інформаційних баз>). Елемент не можна вилучити або змінити. Настройки дати заборони, виконані для цього елемента, будуть використовуватися для всіх користувачів (інформаційних баз), крім тих, які містяться в списку.
      • При необхідності додайте в список потрібних користувачів / групу користувачів (інформаційні бази):
        • Натисніть Підібрати, для того щоб вибрати зі списку кілька користувачів (інформаційних баз).
        • Натисніть Ще - Додати над списком, для того щоб Вибрати користувача / групу користувачів (програму та інформаційну базу для цієї програми. Для кожної програми можна також вибрати елемент <Всі інформаційні бази>).
  • Для того щоб виконати настройку дати заборони, необхідно виділити в списку потрібного користувача/групу користувачів (інформаційну базу) або елемент <Для всіх користувачів> (<Для всіх інформаційних баз>), а самі настройки виконати нижче в полях Спосіб зазначення дати заборони і Дата заборони.

Способи зазначення дати заборони

Програма дозволяє задавати різні Способи зазначення дати заборони змін:

  • Загальна дата – при цьому способі одна дата заборони задається для всіх елементів даних інформаційної бази.
  • По розділах- для кожного розділу програми встановлюється своя дата заборони.
    • Коли дата заборони задається по розділах, стає доступним список, в який можна додавати Розділи (відповідною командою) та вказувати для кожного розділу своє значення дати заборони. Список складається з колонок:
      • Розділ, об'єкт-в даному випадку в колонці перераховуються розділи програми;
      • Дата заборони-виберіть зі списку тип значення дати заборони (можна вибрати довільну дату або одну з відносних дат);
      • Заборона змін за-для довільної дати напишіть або виберіть значення, для відносної дати значення проставляється програмою автоматично.
    • У списку за замовчуванням присутній елемент <Загальна дата>. Значення дати заборони для цього елемента буде використовуватися для всіх розділів за замовчуванням, крім тих, які вже містяться в списку.
    • Якщо в полі Установка дати заборони вказано Для всіх користувачів (для всіх інформаційних баз), то вилучити елемент < Загальна дата> і розділи не можна, при цьому по умовчанню доступні всі розділи, їх можна тільки Змінити-вказати Дату заборони.
    • Якщо в полі Установка дати заборони зазначено По користувачам (по інформаційних базах), то вилучити елемент < Загальна дата> і розділи можна. Якщо дата заборони змін встановлюється для певного користувача/групи користувачів (інформаційної бази), то необхідно вибрати один або кілька розділів.
      • Для додавання розділу виконайте команду Розділи, виберіть потрібні параметри, а потім натисніть кнопку ОК.
  • По об'єктах – дати заборони задаються в розрізі об'єктів програми. Для кожного об'єкта встановлюється своя дата заборони. Якщо вибрано це значення, то стає доступним список, в який можна додавати об'єкти (відповідною командою) і вказувати для кожного об'єкта своє значення дати заборони. Так само, як і в попередньому способі, в списку для аналогічних цілей є елемент < Загальна дата>.
  • За розділами і об'єктами - дати заборони задаються і в розрізі розділів, і в розрізі об'єктів.
    • При цьому стає доступним список, в який можна додавати розділи та об'єкти, вказуючи для кожного розділу і кожного об'єкта своє значення дати заборони. Так само, як і в попередніх способах, у списку міститься елемент <Загальна дата> для аналогічних цілей.
      • Для вибору розділів виконайте команду Розділи.
        • Для вибору декількох об'єктів у виділеному розділі виконайте команду Підібрати.
        • Також в потрібному розділі можна виконати команду Додати контекстного меню по правої кнопки миші, потім вказати об'єкт за допомогою кнопки Вибрати. Також можна скористатися аналогічною командою меню Ще.

Для того щоб встановити дату заборони зміни потрібного розділу (об'єкта) або змінити загальну дату, у потрібному рядку в колонці Дата заборони виконайте команду Змінити контекстного меню по правій кнопці миші або подвійним натисненням на ній кнопки миші. Для завдання дати заборони натисніть кнопку Вибрати. Після вибору дати для повернення до подальших настройок натисніть кнопку ОК.

Дата заборони

  • При установці дати заборони можна задавати не тільки довільну дату (в цьому випадку дату можна написати або вибрати з календаря, натиснувши ), але і відносну.
  • Значення відносних дат:
    • Кінець минулого року;
    • Кінець минулого кварталу;
    • Кінець минулого місяця;
    • Кінець минулого тижня;
    • Попередній день.
  • Для відносних дат (крім значення Попередній день) можна задати кількість днів «відстрочки» до установлення дати заборони, протягом яких зміна даних все ще буде дозволено.
    • Для цього необхідно включити прапорець Дозволити зміну даних до дати заборони. Потім встановіть кількість днів "відстрочки", наприклад, протягом 10 днів після дати заборони.

Звіти за датами заборони

Звіти доступні за відповідною командою. У звітах можливе групування даних:

  • за розділам/об'єктами: виберіть звіт Дати заборони по розділах і об'єктах для користувачів (Дати заборони по розділах і об'єктах для інформаційних баз);
  • за користувачами (за інформаційними базами): виберіть звіт Дати заборони за користувачами (Дати заборони за інформаційними базами).

Звіти дозволяють побачити результуючі дати заборони, встановлені для користувачів/груп користувачів (інформаційних баз) з урахуванням пріоритетів.
