Призначена для створення і зберігання умов автоматичного розподілу вхідної та вихідної кореспонденції по папках. Правила налаштовуються для кожного Облікового запису електронної пошти.

Настройка правила
  • В полі Умова необхідно вибрати найменування створеного правила - текстовий опис умови, за якою будуть відбиратися листи.
    • Текст може бути сформований автоматично, виходячи з налаштованих умов.
    • Виберіть найменування зі списку, після того як будуть сформовані умови правила.
    • Якщо таке найменування занадто довго, можна написати своє найменування. Поле є обов'язковим для заповнення.

  • В полі Поміщати в папку вкажіть, в яку папку потраплять листи, що задовольняють заданим умовам. Для цього виберіть папку зі списку. Поле є обов'язковим для заповнення.

Введення умов

Умови розміщуються в списку, в якому вказуються поля, за якими відбуватиметься відбір, значення полів відбору, при необхідності групуються поля відбору.

  • Задати нову умову можна по команді Додати новий елемент. Умов може бути кілька.
  • У колонці Поле виберіть поле електронного листа, за яким буде проводитися відбір. Для цього вкажіть потрібне поле, потім натисніть кнопку Вибрати.
  • У колонці Вид порівняння можна вибрати операцію порівняння, при цьому крім стандартних операцій, наприклад, Більше або дорівнює, є нестандартні, наприклад, В списку.
    • За допомогою операції порівняння В списку можна задати список можливих значень для відбору, наприклад, сюди можна помістити кілька поштових скриньок партнера, з яких можлива відправка кореспонденції. Вид порівняння Не у списку є запереченням попередньої. Наприклад, в іншу папку потрібно помістити інші листи, крім листів, отриманих від перерахованих в списку відправників.
    • За допомогою операції Містить або Не містить можна використовувати пошук по частині поля (поєднання букв) - найменування, теми, адреси, якщо повністю це поле невідоме. Наприклад, можна задати шаблон поштових скриньок контактних осіб певного партнера, якщо назва сайту партнера відома. В цьому випадку листи з поштових адрес з таким поєднанням букв будуть поміщені в одну папку.
    • Визначити, чи є в полі значення, можна за допомогою операції Заповнено або Не заповнено. Наприклад, за допомогою такої умови можна в окрему папку помістити листи, в яких не вказана тема.
  • Потім налаштуйте Значення. Для цього в залежності від вибраного поля і виду порівняння введіть значення, або натисніть кнопку Вибрати і виберіть із запропонованого списку значень потрібне, або створіть свій список значень, наприклад, список з декількох адрес електронної пошти партнера.
    • Як значення можна використовувати поля документів Електронний лист. Для використання цієї можливості встановіть прапорець Ще - Використовувати як значення поле. Після цього можна Вибрати потрібне поле зі списку.
  • За допомогою прапорця Ще - Докладно можна додати в список умов додаткові колонки:
    • Режим відображення значення, за замовчуванням встановлений Швидкий доступ. Можна також вибрати Звичайний або Недоступний.
    • Представлення для колонки Значення.

Групування умов

  • Можна використовувати кілька умов для більш тонкого відбору. Це дозволить полегшити роботу з поштою, якщо на поштову скриньку компанії надходить багато листів.
    • У цьому випадку за командою Згрупувати умови до виділених умов необхідно застосувати логічні операції І, АБО, Не. Наприклад, для того щоб в папку певного партнера потрапляли його листи, можна створити правило для папки партнера, а в умовах вказати всі адреси електронної пошти партнера і об'єднати умови в групу по логічній операції Або.
    • Якщо відомо, що листи від даного клієнта приходять тільки певному співробітнику або з певних адрес, то можна створити більш точну умова для відбору пошти також за допомогою логічної операції Або.
    • Для того щоб використати групування умов, виділіть одну з потрібних умов, виконайте команду Згрупувати умови, виберіть Тип групи - І, Або, Не. За допомогою лівої кнопки миші перетягніть інші умови в групу або створіть їх усередині групи.
    • Для того щоб Додати нову групу, виконайте відповідну команду меню Ще.
    • Скасувати групування можна за допомогою команди Ще - Розгрупування.
    • Поміняти порядок умов у списку можна за допомогою команд Перемістити вверх і Перемістити вниз меню Ще.
    • Якщо у правилі задається кілька умов без групування, то за замовчуванням до таких умов застосовується логічна операція І.
  • Натисніть кнопку Записати і закрити, після цього створені правила почнуть застосовуватися для всіх нових листів облікового запису (при отриманні листів і при зміні статусу листів, наприклад, з нового на чернетку і т.д.). Скористайтеся відповідною командою в списку правил обробки електронної пошти, для того щоб застосувати правила до старих листів.
