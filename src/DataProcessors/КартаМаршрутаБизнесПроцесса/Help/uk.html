Відкривається зі списку Мої бізнес-процеси або з вікна бізнес-процесу (наприклад, Завдання) по кнопці Карта маршруту.

Хід виконання бізнес-процесу в програмі наочно представлений у виді карти маршруту. Карта маршруту описує логіку бізнес-процесу і весь його життєвий цикл у виді схематичного зображення послідовності проходження точок маршруту. Точка маршруту відображає етап бізнес-процесу, пов'язаний з виконанням, як правило, однієї операції. При переході бізнес-процесу в яку-небудь точку маршруту, що передбачає дії конкретного виконавця, формується задача. Після того як виконавець позначить задачу як виконану, бізнес-процес автоматично перейде до наступної точки маршруту. Таким чином, за допомогою виконання задач бізнес-процес просувається до точки свого завершення.

При роботі з програмою передбачена можливість відображення актуальної карти маршруту для конкретного бізнес-процесу, з урахуванням пройдених і активних точок маршруту. Пройдені (виконані) точки маршруту позначаються штрихуванням.

У кожній точці маршруту доступно одна або кілька дій, що змінюють стан бізнес-процесу.

Дія Хто виконує Виконання команди
Прийняти до виконання Виконавець бізнес-процесу, вказаний в якості виконавця для першої точки маршруту.

Виконавець може прийняти до виконання задачу, призначену на дану роль. Тим самим, задача ховається зі списків задач інших виконавців цієї ролі.

Створити на підставі

Співробітник, який має доступ для роботи з бізнес-процесом.

За допомогою цієї команди можна створити підпорядковані бізнес-процеси.

Виконано

Виконавець завдання, вказаний в якості виконавця для першої точки маршруту Виконати.

Після натискання кнопки Виконано задача змінює свій статус, переходить в список виконаних задач і стає неактивною у списках.

Перевіряючий завдання, вказаний в якості виконавця для другої точки маршруту Перевірити.

Перевіряючий завдання підтверджує виконання, натиснувши кнопку Виконано. Можна написати коментар про виконання завдання. Після цього задача стає неактивною у списках.

Скасовано

Виконавець завдання, вказаний в якості виконавця для першої точки маршруту Виконати.

Натискання кнопки Скасовано означає невиконання задачі. Задача отримує відповідний статус.

Перевіряючий завдання, вказаний в якості виконавця для другої точки маршруту Перевірити.

Перевіряючий підтверджує невиконання задачі.

Повернуто

Перевіряючий завдання, в якості виконавця для другої точки маршруту Перевірити.

Перевіряючий може повернути завдання для більш якісного виконання виконавцем, натиснувши кнопку Повернуто виконавцю.

Перенаправляти

Виконавець і перевіряючий

За допомогою кнопки Перенаправити задачу з'являється в списку задач іншого виконавця.

Перегляд задач по точці маршруту бізнес-процесу

  • Для того щоб переглянути список задач по точці маршруту бізнес-процесу, натисніть кнопку Показати задачі для точки маршруту. Також для цього можна скористатися подвійним натисненням кнопки миші по точці маршруту.
