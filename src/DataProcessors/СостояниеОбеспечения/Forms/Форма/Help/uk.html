Стан забезпечення

Робоче місце призначене для контролю та коригування графіка відвантаження замовлень, за допомогою виконання індивідуальних рекомендацій для замовлення в цілому або для окремих його рядків.

Стан забезпечення дозволяє зарезервувати номенклатуру замовлення у вільному залишку на складі або встановити дату відвантаження, виходячи з дати очікуваного надходження номенклатури на склад.

Робоче місце використовується в трьох основних режимах: для роботи зі списком замовлень обраного складу та/або менеджера для роботи з замовленням, редагований в даний момент і для роботи з усіма замовленнями, що містять цікаву позицію номенклатури.

Порядок роботи

Використання при безпосередньому редагуванні замовлення


Передбачена можливість візуального відображення стану забезпечення:

  • зелений – повністю забезпечений;
  • жовтий – частково забезпечений;
  • сірий – не забезпечений.


Для кожного товару показується рекомендація щодо вибору оптимального варіанту забезпечення (Дія) відповідно до поточного стану забезпечення.
При наявності рекомендацій в колонці Дії можна виконати їх порядково. Для групового виконання всіх рекомендацій необхідно перейти по гіперпосиланню Дії із замовленням.

Після виконання рекомендацій результат переноситься в замовлення, що редагується, по команді Перенести в документ, при цьому в замовленні можуть бути змінені варіанти забезпечення, дати відвантаження і автоматично розбиті рядки за поточним станом забезпечення.

Можливі наступні рекомендації:

  • Резервувати (стрілка вліво). З'явилася можливість забезпечити потребу для товарів, в яких встановлений варіант дії До забезпечення. Товар надійшов на склад (є у вільному залишку) або було зроблене замовлення постачальнику (замовлення на збирання, замовлення на переміщення), товар по якому ще не розподілений в інших замовленнях клієнтів;
  • Резервувати раніше (стрілка вліво). Товар розміщений у замовленні постачальнику (замовленні на переміщення, замовленні на збирання), встановлений варіант Резервувати до дати. З'явилася можливість зарезервувати товар на складі (є вільні залишки) або розмістити в замовленнях з більш ранньою датою надходження. Виконання рекомендації зменшує дату відвантаження по товару, наближаючи її до бажаної дати відвантаження;
  • Резервувати пізніше (стрілка вправо). Товар можна зарезервувати пізніше відповідно до бажаної дати відвантаження. Встановлений варіант дії Резервувати на складі або Резервувати до дати резервує товар раніше бажаної дати відвантаження. Виконання команди дозволяє збільшити дату відвантаження по замовленню, звільнивши товар у ранні дати для резервування і відвантаження по інших замовленнях;
  • Виправити забезпечення (стрілка вправо зі знаком оклику). Товар був зарезервований на складі (варіант дії Резервувати на складі) або товар був розміщений у замовленні постачальнику (Резервувати до дати відвантаження). Однак графік поставки товарів зірвано (товар вчасно не надійшов, на складі немає достатньої кількості). При виконанні даної рекомендації буде змінено варіант забезпечення та змінено дату відвантаження відповідно до поточного стану забезпечення товару. Тобто фактично відбудеться автоматичне перезаповнення варіанту дії і дати відвантаження.

При встановленні для рядків замовлення варіанту дії До забезпечення (Забезпечувати відокремлено) контролюються умови оплати по замовленню. Якщо в умовах оплати по замовленню були присутні етапи з типом Аванс (до забезпечення), то до поточного моменту суми за цими етапами повинні бути оплачені клієнтом.

При проведенні замовлення формується розпорядження службі логістики (постачання) по тих позиціях, для яких встановлено варіант Дії До забезпечення (Забезпечувати відокремлено).
Для контролю дефіциту товарів службою логістики призначене робоче місце Формування замовлень за потребами і звіт Товарний календар.

Використання для обробки переліку замовлень (виклик з панелі навігації розділу)

В даному режимі, робоче місце відображає інформацію щодо забезпечення для переліку замовлень.

Для обмеження списку відображуваних замовлень необхідно використовувати відбори, по складу, менеджеру, номенклатурі. За командою Настроїти список надається розширений список відборів.


Стан забезпечення також може бути використано для обробки виділених замовлень зі списків документів (замовлення клієнтів, замовлення на переміщення і т.п.).

Аналіз стану забезпечення замовлень по окремій номенклатурі

При установці відбору за номенклатурою в списку будуть показані тільки замовлення, що містять відібрану позицію номенклатури. При цьому рядки по даній позиції будуть відображені, а інші рядки будуть приховані.

Відобразити приховані рядки замовлення можна, перейшовши за гіперпосиланням Показати всі позиції.

Режим відбору за номенклатурою не призначений для виконання рекомендацій щодо забезпечення.
Виконання рекомендацій можливе без відборів або з відборами за складом та/або менеджером.
