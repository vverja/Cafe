Призначена для зіставлення даних двох програм.

Список відкривається по команді Виконати зіставлення на сторінці Зіставлення даних Помічника інтерактивної синхронізації даних. Можна також двічі клацнути мишею по рядку, в якому виявлені проблеми зіставлення даних.

Список складається з двох колонок, кожна з яких відповідає інформаційній базі, що бере участь в обміні. Дані згруповані за об'єктами програми (документи, списки). У нижній частині списку виводиться інформаційний рядок: скільки елементів зіставлено, скільки не зіставлено.

В поле Виводити можна вибрати, які дані показувати в списку. По умовчанню виводяться Незіставлені дані.

Зіставлення об'єктів

  • Натисніть Зіставити автоматично (рекомендується), виберіть поля для зіставлення з допомогою прапорців. Деякі поля вибрані програмою по умовчанню. Для того щоб підтвердити свій вибір, натисніть Виконати зіставлення. Після пошуку програма виводить на перегляд зіставлені нею дані. Для підтвердження натисніть Застосувати.
  • Після автоматичного співставлення можна об'єкти, що залишилися, зіставити вручну або змінити зіставлення об'єктів. Виділіть об'єкти двох баз, натисніть Скасувати відповідність, для того щоб спробувати зіставити об'єкти вручну, натисніть Встановити відповідність для того щоб зіставити об'єкти.
  • Для підтвердження натисніть Записати і закрити.

Настройка полів таблиці зіставлення

  • Натисніть Колонки, щоб додати поля в колонки списку. За допомогою прапорців можна відзначити додаткові поля, для підтвердження натисніть Застосувати.

Отримання даних з іншої програми

  • Для того щоб отримати дані з іншої програми, натисніть Ще - Завантажити дані з іншої програми.

Порядок зіставлення об'єктів

  • Рекомендується виконувати зіставлення і завантаження даних з урахуванням посилальних зв'язків. Особливо, якщо поле використовується для зіставлення об'єктів.
  • Наприклад, у конфігурації є довідник договорів контрагентів, який підпорядкований довіднику контрагентів. Зіставлення договорів контрагентів виконується за довідником-власником, тобто за довідником контрагентів. Отже, для правильного зіставлення даних необхідно спочатку зіставити і завантажити довідник контрагентів, а потім - довідник договорів контрагентів.
  • В іншому випадку поля таблиці зіставлення можуть містити фіктивні посилання виду:
    <Об'єкт не знайдений> (26:a0b9001b24e002fe11dfba347dd41412).
  • Фіктивне посилання вказує на об'єкт в поточній інформаційній базі, який ще не був завантажений з повідомлення обміну.
