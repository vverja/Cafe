Призначена для контролю і заміни комплекту правил для синхронізації даних.

Відкривається по кнопці Завантажити комплект правил з настройки синхронізації даних.

Настройка синхронізації даних по умовчанню проводиться у відповідності з так званими Типовими правилами обміну. Правила обміну складаються з правил конвертації об'єктів і правил реєстрації об'єктів.

Правила конвертації об'єктів можна відкрити з настройки синхронізації даних по команді Ще - Завантажити правила конвертації об'єктів. Правила конвертації даних використовуються при синхронізації даних двох програм.

Правила реєстрації об'єктів можна відкрити з настройки синхронізації даних по команді Ще - Завантажити правила реєстрації об'єктів.

Джерело правил

  • Для стандартної синхронізації даних між типовими конфігураціями використовуються типові правила З конфігурації. Однак у ряді випадків необхідно використовувати правила З файлу на комп'ютері (наприклад, якщо синхронізація ведеться між базами з нетиповими конфігураціями).
  • За допомогою перемикача можна встановити, звідки брати правила реєстрації об'єктів:
    • З конфігурації (обрані по умовчанню).
    • З файлу на комп'ютері.
  • Натисніть Завантажити, вкажіть шлях до файлу. Використання правил, завантажених з файлу, може привести до помилок при переході на нову версію програми.
  • Нижче виводиться найменування правил, які в даний момент використовуються.

Інформація про комплект правил

Для контролю виводиться інформація про правила:

  • Попередження про можливість виникнення помилок при використанні правил із зовнішнього файлу;
  • Правила реєстрації об'єктів цієї інформаційної бази - вказується версія і дата створення правил;
  • Правила конвертації об'єктів цієї інформаційної бази - вказується версія і дата створення правил;
  • Правила конвертації кореспондента - вказується версія і дата створення правил.

Режим налагодження

  • Якщо перемикач встановлений в положення З файлу на комп'ютері, стає доступним режим налагодження.
  • Натисніть посилання Настройка режиму налагодження.
  • Установіть прапорець Увімкнути режим налагодження. Після цього стають доступними наступні настройки:
    • Підключити зовнішню обробку для налагодження вивантаження – вказується файл зовнішньої обробки, що містить текстовий модуль обробників вивантаження, сформований в конфігурації "Конвертація даних". У клієнт-серверному режимі роботи вказується шлях на сервері.
    • Підключити зовнішню обробку для налагодження завантаження – вказується файл зовнішньої обробки, що містить текстовий модуль обробників завантаження, сформований в конфігурації "Конвертація даних". У клієнт-серверному режимі роботи вказується шлях на сервері.
    • Режим протоколювання обміну даними - якщо включено, то в процесі обміну даними події, що виникають і помилки будуть записуватися в зазначений файл. Якщо файл не вказано, то повідомлення будуть виводитися в Журнал реєстрації.
    • Не зупиняти виконання обміну даними при виникненні помилки – якщо встановлено, то виконання синхронізації даних не буде зупинено при виникненні будь-якої помилки. Обмін даними завершиться до кінця з виведенням налагоджувальних повідомлень у файл протоколу обміну або журнал реєстрації.

Завантаження правил

  • Встановіть перемикач в положення З файлу на комп'ютері.
  • Натисніть Завантажити.
  • Вкажіть шлях до файлу з правилами.

Вивантаження правил

  • В разі потреби правила реєстрації можна вивантажити в файл.
  • Для цього натисніть Ще - Зберегти правила в файл. Виберіть шлях до файлу на комп'ютері або на мережевому диску, найменування проставляється програмою по умовчанню. Натисніть Зберегти.

Заміна типових правил конвертації об'єктів правилами із зовнішнього файлу

  • Встановіть значення перемикача в положення З файлу на комп'ютері і по кнопці Завантажити виберіть файл правил на комп'ютері або в мережевій папці.
  • Зверніть увагу, що завантажені коректні правила, які підходять для використовуваних версій Програми-джерела і програми-приймача. Для цього виводиться Інформація про комплект правил, де зазначено, від якої дати ці правила.
  • Натисніть на кнопку Записати і закрити. Завантажені правила будуть застосовані негайно.

Примітка: правила конвертації об'єктів і правила реєстрації об'єктів використовуються для всіх однотипних синхронізацій даних (які ґрунтуються на одному плані обміну; наприклад: на базі плану обміну Повний налаштовані дві синхронізації даних "Синхронізація з філією в Чернігові" і "Синхронізація з представництвом у Києві"). Тому якщо в програмі налаштовано кілька однотипних синхронізацій, то заміну правил досить виконати одноразово (вибравши будь-яку з налаштованих синхронізацій даних).
