Бюджетирование и планирование

Назначение подсистемы

Подсистема предназначена для многовариантного (многосценарного) планирования денежных средств и финансовых показателей хозяйственной деятельности на сроки, как правило, не менее месяца. Целью подсистемы является организация процессов управления через бюджетные формы, данными которые позволяют получить четкую картину как текущей, так и будущей финансовой ситуации на предприятии с точки зрения оценки выбранных показателей (прибыли, остатков денежных средств и т.д.).
Подсистема позволяет решать следующие задачи:

  • планирование финансового положения предприятия на заданный период времени;
  • планирование потоков денежных средств;
  • составление основных бюджетов предприятия:
  • бюджет доходов и расходов;
  • бюджет движения денежных средств;
  • прогнозный баланс;
  • составление операционных и функциональных бюджетов, а так же сводного финансового плана;
  • моделирование будущей деятельности с использованием различных сценариев работы;
  • сравнение данных бюджетных моделей между собой;
  • план-фактный анализ отклонений в разрезе выбранных аналитик;
  • получение информации для системы оценки и мотивации персонала;
  • управление процессами составления и утверждения бюджетов, а так же анализа результатов за период.

Настройка подсистемы

Для настройки подсистемы в разделе НСИ и администрирование - Бюджетирование и планирование необходимо осуществить следующие настройки:

  • Для возможности использования функционала по финансовому планированию установить функциональную опцию Бюджетирование.
  • Для возможности управления регламентом подготовки и контроля своевременности исполнения бюджетов ответственными лицами установить функциональную опцию Использовать оповещения для задач бюджетирования.
  • Для возможности автоматического расчета хранимых данных бюджетирования в момент проведения документов (формирования отчетов) установить функциональные опции Автоматически запускать фоновое задание расчета хранимых данных при проведении документов и Автоматически запускать фоновое задание расчета хранимых данных из отчетов.

Работа подсистемы

Начальная работа с подсистемой Бюджетирования осуществляется по средствам подготовки нормативно-справочной информации: Статей бюджетов, Видов бюджетов, Сценариев, а так же используемых в них аналитик.

На основании заполненых в справочниках данных осуществляется создание Модели бюджетирования, которая в свою очередь определяет:

  • Использование основных аналитик бюджетов: Организации и Подразделения.
  • Правила заполнения плановых показателей и статей.
  • Этапы бюджетного процесса, задающие регламент планирования и контроля по используемым в модели видам бюджетов.

По видами бюджетов может быть оформлена различная отчетность, для отображения которой используется отдельное рабочее место Бюджетные отчеты. Отчеты могут быть представлены в виде трех типов:

  • Бюджет – выводит только плановые значения;
  • Сравнение сценариев – позволяет сравнивать разные планы и контролировать отклонения по статьям;
  • План-фактный анализ – позволяет сравнивать выбранный сценарий бюджетирования с фактическими данными, полученными за период.

По одному из настроенных видов бюджетов и сценарию (например, оптимистическому) осуществляется формирование финансовых планов - Экземпляров бюджета. Документ Экземпляр бюджета является основным инструментом для установки значений по статьям бюджетов и их аналитикам. Экземпляр бюджета формируется в виде отчета при заполнении значений строк автоматически пересчитывает итоги по группам статей, связанным статьям и показателям.

Для возможности управления последовательностью создания бюджетов в подсистеме используется Бюджетный процесс (настраиваемый в рамках Модели бюджетирования), который определяет последовательность Этапов бюджетного процесса с заданными порядком исполнения по датам начала работ, а так же срокам, установленным для составления и утверждения бюджетов и проведения план-фактного анализа. На основе расписания, заданного в шагах процесса, системой автоматически создаются Задачи бюджетного процесса на соответствующих исполнителей.
Шаги бюджетного процесса группируются в этапы процесса, которые задают способ выполнения задач: последовательно или параллельно.

В случае применения подхода к настройке бюджетирования «от простого к сложному», когда на начальном этапе она осуществляется централизовано плановым отделом, процедура построения этапов бюджетного процесса является опциональной и может не выполняться.

Участники бюджетного процесса выполняют задачи и указанные в них действия:

  • заполняют планы с помощью документов Экземпляр бюджета,
  • анализируют плановые и полученные по факту данные в Бюджетных отчетах,
  • утверждают планы или вносят в них корректировки.

Контроль над выполнением текущих задач и оценкой времени, необходимого для исполнения последующих задач (с учетом сроков переноса) осуществляется при помощи рабочего места Монитор бюджетных процессов. Особенно актуально использование данного рабочего места при использовании большого количества видов бюджетов и сотрудников, вовлеченных в процесс их заполнения.

Анализ данных

Анализ данных подсистемы бюджетирования может быть осуществлен при помощи следующих основных отчетов:

  • Статистика по выполнению задач бюджетного процесса.
  • Оборотная ведомость по статьям бюджетов.
  • Оборотно-сальдовая ведомость по показателям бюджетов.
