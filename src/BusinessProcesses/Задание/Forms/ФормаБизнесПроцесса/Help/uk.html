Призначено для постановки та перевірки виконання довільних задач.

Можна запустити з документа програми по команді Створити на підставі (з бізнес-процесу, при відповідних настройках - таке завдання стане підпорядкованим бізнес-процесом). Також можна почати новий бізнес-процес Завдання по команді Створити в списку Завдання.

Відомості про завдання

  • Головне завдання - заповнюється програмою автоматично, якщо бізнес-процес є підпорядкованим. За посиланням у цьому полі можна переглянути дані про головний бізнес-процес.
  • Завдання - напишіть заголовок завдання, який буде показаний в списках бізнес-процесів. Поле є обов'язковим для заповнення.
  • Важливість - приймає значення Звичайна (по умовчанню), Низька, Висока. Для того щоб виділити бізнес-процес, або навпаки, вказати, що виконання цього бізнес-процесу може бути відкладено, виберіть відповідне значення цього поля.
    • Виконавець-виберіть виконавця зі списку користувачів програми або зі списку виконавців. Поле є обов'язковим для заповнення.
    • Строк - вкажіть дату і час, коли потрібно виконати завдання.
    • Предмет - автоматично проставляється документ програми, на підставі якого створено завдання.
    • У текстовому полі нижче можна при необхідності більш детально пояснити суть завдання. Недоступно для підпорядкованих бізнес-процесів.
    • Увімкніть прапорець Перевірити виконання (по умовчанню включений), для того щоб:
      • призначити перевіряючого зі списку користувачів;
      • ввести Строк - дату і час, до якого треба перевірити виконання завдання. Якщо строк не заповнений, то перевірка не проводиться.
      • Автор - автоматично проставляється програмою поточний користувач. Автора можна призначити зі списку Користувачі.
      • Розпочато - дата введення завдання. Заповнюється програмою автоматично. Поле не підлягає редагуванню.

      Старт бізнес-процесу

      • Після того як завдання заповнено, натисніть кнопку Стартувати і закрити, для того щоб зберегти Завдання, почати бізнес-процес і повернутися в програму.
      • Для того щоб почати бізнес-процес, можна також скористатися командою Старт. В цьому випадку програма збереже завдання, почне бізнес-процес, але завдання залишиться відкритим.
      • Програма автоматично формує задача, з яких складається бізнес-процес Завдання:
        • виконання завдання;
          • після виконання цієї задачі в залежності від настройок завдання стає виконаним або формується задача перевірка виконання.
      • Для того щоб записати чернетку завдання, але не стартувати його, натисніть Записати. Таке завдання можна дозаповнити і стартувати пізніше, в більш зручний момент часу зі списку завдань або відкривши завдання.

      Управління рухом бізнес-процесу

      • Для того щоб Зупинити або Продовжити бізнес-процес, скористайтеся відповідними командами (активна одна з команд). Зупинений бізнес-процес у списку виділяється значком .

      Перегляд карти маршруту

      • Натисніть кнопку Відкрити карту маршруту бізнес-процесу, щоб переглянути бізнес-процес у наочній формі.
      • Також можна скористатися командою Ще - Карта маршруту.
      • Карта маршруту описує логіку бізнес-процесу і весь його життєвий цикл у вигляді схематичного зображення послідовності проходження точок маршруту.

      Перегляд задач

      • Для того щоб переглянути задачі, з яких складається даний бізнес-процес, виконайте команду Задачі в панелі навігації. Якщо завдання не стартовано, то список задач порожній.
