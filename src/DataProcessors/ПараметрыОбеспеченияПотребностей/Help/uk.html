Параметри забезпечення потреб

Робоче місце призначене для установки способів і методів забезпечення номенклатури. Можливість використання робочого місця визначається функціональною опцією Забезпечення потреб-Розширене в розділі НДІ і адміністрування–Склад і доставка-Забезпечення потреб.

Порядок роботи по установці параметрів забезпечення:

  • Установка фільтра за складом (обов'язковий).
  • Відбір номенклатури для якої потрібне встановлення параметрів забезпечення (команда Заповнити список товарів).
  • Визначення способів і методів забезпечення для рядків таблиці (можлива установка методу і способу відразу для декількох рядків, для цього їх потрібно виділяти, утримуючи клавішу Сtrl на клавіатурі, або виділити всі рядки по комбінації клавіш Сtrl+A).
  • Крім заповнення способів і методів забезпечення - можна задати однакову кількість запасу для різних рядків-мінімальне, максимальне, страхове; можливо також задати норму споживання для рядків з відповідним методом забезпечення.

Методи забезпечення

Методи забезпечення визначають в який момент часу програма буде пропонувати сформувати замовлення по номенклатурі і яку кількість буде запропоновано до замовлення:

  • Замовлення під замовлення - при даному методі забезпечення до замовлення буде пропонуватися тільки кількість, необхідна для відвантаження за замовленням.
  • Підтримка запасу (min-max) - при даному методі необхідно вручну задати мінімальний і максимальний запаси товару. При зниженні залишку товарів до мінімального запасу програма буде пропонувати до замовлення кількість, що дорівнює максимальному запасу. Статистика продажів при цьому методі не враховується.
  • Підтримка запасу (розрахунок за статистикою) - при цьому методі буде запропоновано сформувати замовлення, коли поточного залишку товарів (за вирахуванням страхового) буде вистачати не більше, ніж на строк виконання замовлення за способом забезпечення. При цьому денне споживання буде розраховане за статистикою минулих періодів. Кількість до замовлення за цим методом розраховується таким чином, щоб товару вистачило на заданий строк забезпечення або до зазначеної дати.
  • Підтримка запасу (розрахунок за нормою) - при даному методі буде запропоновано сформувати замовлення, коли поточного залишку товарів (за вирахуванням страхового) буде вистачати не більше, ніж на строк виконання замовлення за способом забезпечення, заданим для даного товару. Денна норма споживання при цьому методі задається вручну. Кількість до замовлення за цим методом розраховується таким чином, щоб товару вистачило на заданий строк забезпечення або до зазначеної дати.

При методах Підтримання запасу (..), якщо кількість, необхідна для відвантаження за замовленнями, буде перевищувати кількість, необхідну для підтримки складського запасу, автоматично буде запропоновано замовити кількість, необхідну для відвантаження за замовленнями.

Страховий запас задається тільки при методах забезпечення Підтримання запасу(..). Страховий запас дозволяє покрити можливі сплески попиту, в порівнянні з розрахованою статистикою, або забезпечити запас товарів при зриві строку поповнення запасу.

Програма може рекомендувати метод забезпечення Підтримання запасу (розрахунок за статистикою), якщо статистика витрат товару відома і відхилення від розрахованого споживання в різні дні невелика. Середньоденне споживання, відхилення споживання і статистику - можна побачити в крайніх правих колонках таблиці. Також за допомогою фільтра Рекомендація можна відібрати тільки рядки, за якими рекомендується встановити метод Підтримання запасу (розрахунок за статистикою)

При підтримці запасу враховуються також потреби по замовленнях до відвантаження. Такі потреби фіксуються в замовленнях клієнтів і внутрішніх замовленнях за рахунок установки для замовлених позицій дії До забезпечення або Забезпечувати відокремлено. Настройка Потреби замовлень до відвантаження визначає яку кількість буде запропоновано до замовлення в залежності від сумарних потреб по замовленнях до відвантаження.

  • Забезпечувати за рахунок запасів-якщо кількість товарів у замовленнях з дією До забезпечення перевищує величину, що замовляється для підтримання запасу, то замовляється максимальна з двох величин.
  • Забезпечувати незалежно від підтримання запасів - кількість до замовлення складається з кількості для підтримання запасу і кількості в замовленнях з дією До забезпечення.

Відокремлені замовлення завжди забезпечуються незалежно від підтримання запасів.

Додаткові фільтри для відбору рядків

Над таблицею, під відбором по складу, розташовані додаткові фільтри для настройки рядків.
Фільтр за характеристиками дозволяє:

  • показати тільки рядки по номенклатурі в цілому (Приховати характеристики)
  • приховати рядки за номенклатурою в цілому, за якою ведеться облік за характеристиками (Приховати )
  • приховати номенклатуру, за якою не ведеться облік за характеристиками і рядки за номенклатурою в цілому (Тільки за характеристиками)
  • показати лише номенклатуру, за якою ведеться облік за характеристиками (За характеристиками та )
  • показати тільки номенклатуру, за якою ведеться облік за характеристиками, без характеристик цієї номенклатури (Тільки )
  • показати тільки номенклатуру, за якою не ведеться облік за характеристиками (Тільки )
