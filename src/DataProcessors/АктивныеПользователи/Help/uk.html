Містить список користувачів, які працюють в даний момент часу в програмі.

У цьому списку відображаються не тільки користувачі, що працюють з інформаційною базою за допомогою програм "Тонкий клієнт", "Веб-клієнт" або "Конфігуратор", але й такі користувачі як "COM-з'єднання", "WS-з'єднання", "Фонове завдання" і т. і.

Список виводиться по відповідній команді розділу Адміністрування - Підтримка та обслуговування.

В списку виводяться:

  • Користувач - повне ім'я користувача. Відображаються також зовнішні користувачі;
  • Додаток - зазначається Тип додатка, наприклад, Тонкий клієнт. Режим підключення також зазначається. Відображаються також сеанси, встановлені в інших режимах підключення, наприклад, з'єднання (COM) або веб-сервіси;
  • Початок роботи - дата і час початку роботи;
  • Комп'ютер - мережеве ім'я комп'ютера, з якого виконаний вхід в програму;
  • Сеанс - номер сеансу, а також поділ даних.

В нижньому рядку списку підраховується загальна кількість активних користувачів.

Жирним шрифтом виділено рядок, що відповідає тому сеансу роботи користувача, з якого відкрито список активних користувачів.

Для зручності перегляду списку можна використовувати сортування по будь-якій колонці.

Відбори

  • За допомогою поля Показувати можна вибрати зі списку відбір по одному з напередвизначених значень. По умовчанню вибрано Всі сеанси.

Перегляд користувача

  • Для того щоб переглянути відомості про користувача, скористайтеся подвійним натисненням кнопки миші по потрібному рядку.
  • Також можна виділити потрібний рядок і виконати команду Ще - Відкрити.

Перегляд журналу реєстрації

  • Для аналізу роботи обраного користувача за командою Журнал реєстрації можна перейти до журналу реєстрації, відфільтрованого за обраним користувачем.

Оновлення списку

  • Список активних користувачів не є динамічним списком. Для того щоб побачити активних користувачів, які потрапили в список після його відкриття, натисніть Оновити поточний список або клавішу F5.

Завершення сеансу (клієнт-серверний режим)

  • У клієнт-серверному режимі роботи доступна команда Завершити сеанс, примусово завершальна вибрані в списку сеанси користувачів.
  • Для коректної роботи цієї команди необхідно попередньо задати параметри адміністрування програми (вікно настройок відкривається автоматично).
