Способы обеспечения потребностей

Справочник предназначен для регистрации способов обеспечения потребностей, определяющих каким образом и из каких источников будет удовлетворяться потребность в номенклатуре.
Способ обеспечения определяет будет номенклатура покупаться, перемещаться с другого склада, производиться, или получаться с помощью операции сборки/разборки. Также способ определяет когда и в каком количестве система будет предлагать обеспечить потребности. Возможность использования способов обеспечения определяется функциональной опцией Расширенное в разделе НСИ и администрирование – Склад и доставка - Обеспечение потребностей.

Типы обеспечения
  • Если номенклатура закупается у поставщика, то следует указать тип обеспечения Покупка. Для такого способа можно уточнить конкретного поставщика, соглашение и условие цены, которые будут автоматически подставлены в заказ поставщику, сформированный по данному способу обеспчения. Также в обработке Обеспечение потребностей для закупаемых товаров будет показываться цена по указанному соглашению с поставщиком.
  • Если товар поступает на склад путем перемещения с другого склада, то следует указать тип обеспечения Перемещение. Опционально можно указать конкретный склад, с которого товар будет перемещаться.
  • Если товар поступает на склад после проведения операции сборки (разборки), то следует указать соответствующий тип обеспечения.
  • Если товар производится силами переработчика, то следует указать тип обеспечения Производство силами стороннего переработчика.


Для всех способов обеспечения опционально можно указать организацию и подразделение. Организация будет использована для заполнения информации в заказе, созданном по данному способу. Подразделение указывает какое из подразделений предприятия осуществляет поставку по данному способу и, соответственно, должно обеспечивать потребности в номенклатуре, обеспечиваемой данным способом.

Пример: Компания занимается продажей компьютерной техники. По способу обеспечения "Покупка у дистрибьютора" все потребности в товарах обеспечиваются закупочным подразделением. По способу обеспечения "Сборка компьютеров" потребности в готовых системных блоках обеспечивает сборочное подразделение, собирающее компьютеры из готовых комплектующих.

Сроки исполнения заказа


В поле Срок исполнения заказа следует указать срок, за который может быть исполнен заказ, созданный по данному способу.

Срок исполнения заказа отсчитывается по рабочим дням календаря предприятия. Например: если сегодня понедельник, а суббота и воскресенье выходные дни по календарю предприятия, и сегодня планируется заказ по способу обеспечения со сроком исполнения 7 дней, то поступление заказа будет запланировано на следующую среду, через 7 рабочих дней.

Определение момента поставки


В способе обеспечения следует задать в какие моменты программа должна предлагать заказывать товары этим способом: при критическом остатке, или по определенному графику.

В варианте Заказ при достижении точки заказа программа будет предлагать к заказу номенклатуру, по которой необходимо сформировать заказы сегодня (чтобы вовремя обеспечить потребности в номенклатуре по текущим заказам или с учетом предполагаемого потребления и текущих остатков на складе ). Количество к заказу определяется как потребность на обеспечиваемый период.


Для товаров обеспечиваемый период рассчитывается по рабочим дням склада, для номенклатуры с типом работа - по рабочим дням предприятия. Например, поставка по способу обеспечения возможна 14.01.2013, в понедельник. Если обеспечиваемый период задан 10 дней, а суббота и воскресенье выходные дни - то заказ клиента с датой отгрузки 25.01.2013 попадет в обеспечиваемый период.

В варианте Заказ по графику система будет предлагать формировать заказы по заранее определенному графику, чтобы номенклатура была обеспечена в указанные даты поставки. Количество к заказу рассчитывается таким образом, чтобы его хватило до даты следующей поставки.


При заказе по графику, до даты следующей поставки, для товаров количество к заказу будет рассчитываться по числу рабочих дней склада.

В обоих вариантах, если используется метод обеспечения Поддержание запаса (min - max), то количество к заказу рассчитывается не по дням, а предлагается фиксированное, заданное в максимальном запасе метода обеспечения.

Гарантированный срок отгрузки позволяет обеспечить запас времени относительно срока исполнения заказа. Гарантированный срок отгрузки используется для расчета плановой даты отгрузки необеспеченных позиций заказов.

Использование методов обеспечения


При использовании методов обеспечения Поддержание запаса (расчет ...) расчет количества к заказу по статистике (норме) производится за период, начинающийся от даты поставки по заказу. Потребность по запланированным к отгрузке заказам учитывается по всем неотгруженным заказом, с датой отгрузки до конца обеспечиваемого периода. Если потребность по заказам превышает рассчитанное значение по статистике (норме) - будет предложено заказать количество, необходимое для отгрузки по заказам.

Пример: Срок исполнения заказа по способу обеспечения - 3 дня, обеспечиваемый период - 7 дней. Для номенклатуры, которой задан этот способ, в методе обеспечения указана норма потребления 3 шт/день. На текущий момент имеется просроченный заказ на отгрузку 2 шт. товара и заказ на 20 шт. запланирован к отгрузке через 6 дней.
Рассчитанная потребность по норме потребления: Обеспечиваемый период (7 дней) * Норма потребления (3 шт) = 21 шт.
Заказы учитываются все, до даты окончания обеспечиваемого периода (обеспечиваемый период начинается с даты исполнения заказа, срок исполнения заказа - 3 дня, значит конец обеспечиваемного периода через 3+7 = 10 дней). Таким образом, учитываются заказа на 2 +20 = 22 шт. Общая потребность по заказам = 22 шт.
К заказу по способу обеспечения будет предложено 22 шт.
