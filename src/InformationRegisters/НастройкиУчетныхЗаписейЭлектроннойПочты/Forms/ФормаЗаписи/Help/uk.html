Настройка використання облікового запису у вбудованому поштовому клієнті
  • Настройка проводиться на вкладці Основне за допомогою перемикача Використання для вбудованого поштового клієнта:
    • Виберіть значення Отримувати, зберігати і відправляти листи з цього облікового запису у вбудованому поштовому клієнті, якщо з облікового запису електронної пошти планується листування.
    • Якщо використання у внутрішньому поштовому клієнті не передбачається (наприклад, цей обліковий запис використовується для обмінів з іншими програмами), то виберіть значення Не використовувати.

Призначення прапорця Персональна обліковий запис

  • Настройка проводиться на вкладці Основне.
  • Облікові записи електронної пошти у вбудованому поштовому клієнті діляться на два типу: Персональні та Колективні.
  • Якщо обліковий запис Персональна, то для неї прапорець повинен бути включений.
    • Виберіть Користувача облікового запису (відповідального за обліковий запис) зі списку користувачів. Поле є обов'язковим для заповнення.
    • Для персонального облікового запису відповідальний за неї управляє складом папок і є відповідальним за обробку листів.
  • Для колективного облікового запису відповідальним за папки може бути один користувач, а відповідальним за обробку листів по умовчанню інший (наприклад, адміністратор програми).
    • Виберіть Відповідального за обробку листів зі списку користувачів. Поле є обов'язковим для заповнення.
    • Виберіть Відповідального за ведення папок зі списку користувачів. Поле є обов'язковим для заповнення.

Приклад використання колективного облікового запису

  • Велика компанія має велику кількість клієнтів по всій країні.
  • У компанії є єдина поштова адреса, на який клієнти шлють претензії.
  • Склад і порядок папок для облікового запису визначає керівник служби, яка відповідає за обробку претензій.
  • Відповідальний за первинний розбір листів - це інший співробітник. Він обробляє вхідну кореспонденцію і призначає листам відповідальних, які надалі відповідатимуть за обробку претензії, викладеної в листі.

Призначення прапорця Вилучати листи після відправки

  • Настройка проводиться на вкладці Основне.
  • Якщо немає необхідності зберігати вихідні листи, успішно відправлені з даного облікового запису, то цей прапорець повинен бути включений.
  • У цьому випадку після надсилання листа буде видалено.

Призначення прапорця поміщати листи в папку листи-підстави

  • Настройка проводиться на вкладці Основне.
  • Якщо цей прапорець включений, то для листів, які є відповіддю або пересилається листом, не застосовуватимуться правила обробки.
  • Такі листи завжди будуть поміщатися в ту ж папку, що і лист-підставу.

Увімкнення імені користувача, вказаного в обліковому записі, до представлення відправника листа
  • Встановіть прапорець В представлення включати ім'я користувача на вкладці Основне.

Настройка підпису до листа
  • Можна створити підпис двох видів:
    • для нових повідомлень- на вкладці Підпис для нового.
    • для повідомлень при відповіді або пересиланні- на вкладці Підпис при відповіді.
  • Встановіть прапорець Включити для потрібного виду підпису.
  • Виберіть зі списку формат підпису:
    • Звичайний текст;
    • HTML - створення підпису у форматі HTML дозволить форматувати текст і вставляти в нього картинки.
  • Напишіть підпис у відповідному полі. При виборі формату HTML можна форматувати текст різними способами, вставляти картинки і гіперпосилання. Текст можна форматувати за допомогою відповідних кнопок, а також команд меню Ще.
  • Необхідно враховувати, що при перемиканні раніше введеного підпису з формату HTML у Звичайний текст частину інформації (наприклад, картинки, гіперпосилання та інші елементи, які існують тільки у форматі HTML), вилучається безповоротно. Програма виводить про це повідомлення. Для підтвердження натисніть Так.
  • Також підписи можна настроїти по команді Ще - Персональні настройки зі списку взаємодій.

Важливо. Створення підписів у форматі HTML доступно, якщо це передбачено настройками програми. Цю настройку можна змінити в розділі НДІ і адміністрування - Органайзер, включивши прапорець Вихідні листи у форматі HTML.
