Призначений для ведення груп зовнішніх користувачів.

Зручно використовувати групи, коли в програмі багато зовнішніх користувачів:

  • для групування великої кількості зовнішніх користувачів (наприклад, при виборі зовнішнього користувача);
  • для групової настройки прав доступу для всіх зовнішніх користувачів групи.

Групи створюються пласким списком (в якому групи не залежать один від одного) або ієрархічним. В ієрархічному списку зовнішні користувачі вкладених груп входять у групи вищого рівня (батьківські).

Існує визначена група Всі зовнішні користувачі, яка включає всіх зовнішніх користувачів. Ця група по умовчанню є поточною.

Коли використовується безпосереднє призначення ролей, їх можна призначати не тільки для кожного зовнішнього користувача, але і для групи зовнішніх користувачів.

Групування зовнішніх користувачів

Групи зовнішніх користувачів можуть формуватися, наприклад, в залежності від приналежності до об'єктів програми:

  • Контрагенти
    • Довірені партнери
    • Основні покупці
    • Основні постачальники
    • Учасники опитування
  • Фізичні особи.

Можна створити групи зовнішніх користувачів, які включатимуть учасників одного типу, наприклад, респондентів. Так зручніше налаштувати права доступу.
